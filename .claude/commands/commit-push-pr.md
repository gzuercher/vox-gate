---
description: Änderungen committen, pushen und PR erstellen
---

Führe folgende Schritte aus:

1. Prüfe `git status` und `git diff --stat` für eine Übersicht der Änderungen
2. Erstelle eine präzise Commit-Message auf Deutsch im Imperativ
3. Gruppiere zusammengehörige Änderungen in einen Commit
4. Committe die Änderungen
5. Pushe den Branch zum Remote
6. Erstelle einen Pull Request mit:
   - Titel: Kurzbeschreibung der Änderung (Deutsch)
   - Body: Was wurde geändert, warum, und was muss beim Review beachtet werden

Falls der Branch noch nicht existiert, erstelle ihn mit einem beschreibenden Namen (z.B. `feature/benutzer-validierung`).
