---
description: Commit changes, push, and open a pull request
---

Run the following steps:

1. Check `git status` and `git diff --stat` for an overview.
2. Write a precise commit message in English, imperative mood.
3. Group related changes into a single commit.
4. Commit the changes.
5. Push the branch to the remote.
6. Open a pull request with:
   - Title: short description of the change (English)
   - Body: what changed, why, and what reviewers should pay attention to.

If the branch does not yet exist, create it with a descriptive name
(e.g. `feature/user-validation`).
